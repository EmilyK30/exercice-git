# Mes notes du Lab Git 1

## Ce que j'ai appris aujourd'hui
Aujourd'hui, j'ai appris à utiliser Git et GitHub. Voici ce que je retiens :
### 1. Pourquoi on utilise Git
Avant, je ne comprenais pas pourquoi on utilisait Git. Maintenant je vois le problème : quand on travaille à plusieurs sur un projet, sans Git on finit par avoir des fichiers qui s'écrasent, on ne sait plus qui a fait quoi, et on perd du travail. Git résout tout ça en gardant un historique de toutes les modifications.
### 2. Installer Git sur Windows
J'ai téléchargé Git sur git-scm.com et je l'ai installé. Après l'installation, j'ai configuré mon nom et mon email
### 3. Les trois zones de Git
C'est le point que j'ai trouvé le plus important à comprendre :
- Le répertoire de travail : c'est là où je modifie mes fichiers
- La zone de préparation (staging) : c'est quand je fais git add, je prépare les fichiers avant de les sauvegarder
- Le dépôt : c'est là où Git garde l'historique des commits
### 4. Créer un dépôt et faire des commits
Avec mkdir, cd, git init, git add, git commit
### 5. Pousser sur GitHub
J'ai créé un compte GitHub (EmilyK30) et j'ai créé un dépôt vide. Ensuite j'ai lié mon dépôt local avec :
- git remote add origin https://github.com/EmilyK30/exercice-git.git
- git push -u origin main
